## 1. Проблема и цель

Сайт разъезжается на телефоне, пользоваться невозможно. Цель — сделать **весь
сайт** (все вкладки, включая Admin/Отчёт/импорт Excel) юзабельным на мобильных
ширинах без горизонтального переполнения, не сломав десктоп.

Стек: React + Vite + **Tailwind CSS + inline-стили**. `src/App.jsx` — монолит
6605 строк. `<meta viewport>` есть. Медиа-запросов почти нет; адаптивность
отсутствует — сетки заданы фиксированным числом колонок.

**Ключевой нюанс:** проблемные сетки заданы **inline-стилями**, а inline-стиль
не поддерживает media-queries. Поэтому Tailwind-префиксы `sm:` работают только
там, где уже используется `className`; для inline нужен либо CSS-функция в самом
значении (`min`, `clamp`, `auto-fit`), либо JS-детект ширины.

## 2. Подход (гибрид CSS-first + точечный хук)

- **Один брейкпоинт — 640px** (мобила). Хелпер `useIsMobile()` на
  `window.matchMedia('(max-width: 640px)')` с подпиской на change — только для
  случаев, где CSS-значений недостаточно.
- **CSS-first** (правка прямо в существующих inline-значениях, без media-query —
  основная масса):
  1. **Карточные сетки** `repeat(N, 1fr)` → `repeat(auto-fit, minmax(<min>, 1fr))`
     — сворачиваются автоматически.
  2. **Модалки** фикс-ширины → `width: min(100vw - 32px, <maxWidth>)`.
  3. **Таблицы** → обёртка `<div style={{ overflowX: 'auto' }}>`, у таблицы
     `minWidth`, чтобы колонки скроллились внутри блока, а не ломали страницу.
  4. **Padding контейнера** → `clamp(12px, 4vw, 24px)`.
  5. **Поля/контейнеры с жёстким `minWidth`** → снять или `flex: 1` на мобиле.
- **Tailwind-префиксы** там, где уже `className`: напр. `grid-cols-1 sm:grid-cols-2`.
- **Хук `useIsMobile`** точечно: хедер (сворачивание кнопок), и любые места, где
  требуется иная раскладка, а не просто сворачивание сетки.
- **viewport**: убрать `maximum-scale=1.0` (разрешить пользовательский зум —
  доступность). Оставить `width=device-width, initial-scale=1.0`.

## 4. Критерии приёмки

1. На ширине **360px и 390px** ни один экран не даёт горизонтального
   переполнения `body` (намеренно скроллируемые таблицы/доска — внутри своей
   области, не ломая страницу).
2. Все вкладки (Dashboard, Проекты, Задачи, Финансы, Заказчики, Admin, Отчёт,
   импорт) открываются и читаемы.
3. Модальные окна влезают в экран, не обрезаются по краям.
4. Хедер и ряды фильтров переносятся/сворачиваются, не вылезают за край.
5. **Десктоп-вид (≥640px) визуально не изменился** — регрессии нет.
6. Сборка `npm run build` проходит без ошибок.
7. Ручная проверка в DevTools responsive на 360 / 390 / 768px.

## 5. Вне охвата

- Редизайн/переработка UX (это адаптация существующего, не новый дизайн).
- Декомпозиция монолита App.jsx (отдельная задача; здесь — только хирургические
  правки по месту).
- PWA / офлайн / тач-жесты сверх нативного скролла.
