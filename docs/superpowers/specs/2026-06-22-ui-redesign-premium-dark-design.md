# UI-редизайн «КЛИМАТ-ПРО premium-dark» — дизайн

Дата: 2026-06-22. Стадия: дизайн (brainstorm завершён, одобрен владельцем).
Метод: superpowers:brainstorming (мокап + web-research референсов → решения → дизайн-система → approval).
Следующий шаг: writing-plans → реализация по фазам.

## 1. Контекст и цель

Текущий UI дашборда — «строго и просто»: тёмная тема (`#0a0a0a/#141414`), золото `#d4af37`, inline-стили,
lucide-иконки, шрифт **Geist Variable** (уже подключён), **framer-motion** (уже в зависимостях, есть
stagger-входы на дашборде + layoutId tab-indicator). Владелец хочет поднять визуал до уровня Linear/Vercel/
Raycast («можно жертвовать весом ради красоты»). Мокап (premium-dark) одобрен как направление.

**Референсы:** Linear/Vercel/Raycast (эталон dashboard+motion); Awwwards dark-mode/black коллекции;
Flux Dashboard (frosted-glass header, dot-grid, слоёные цветные тени, Cmd+K, AnimateNumber, pill-формы).

## 2. Решения владельца

1. **Aurora-фон — только золотое** свечение (убрать бирюзу/фиолет), приглушённое + dotted-grid + виньетка.
2. **Cmd+K command palette — да** (поиск/переход по проектам/задачам/заказам/разделам).
3. **Все эффекты:** spotlight-hover карточек, count-up чисел, слоёные цветные (золотистые) тени + frosted-glass
   шапка, magnetic-кнопки, scroll-reveal секций, hover-lift, shimmer-скелетоны.
4. **Объём — весь сайт** (применить дизайн-систему ко всем экранам).
5. **Шрифт** — Geist (есть), усиленная иерархия; новый display-шрифт по умолчанию НЕ тащим (бережём вес;
   добавим точечно, если на реализации захочется).

## 3. Принцип «pro vs китч» (правило дизайн-системы)

Из motion-гайдов (Linear/Motion): **сдержанность** отличает профи. Тайминги: hover 150-220ms, входы
250-400ms, exit быстрее. Amplitude: сдвиги 8-12px (НЕ 20), hover-scale 1.02 (НЕ 1.2). Spring для интерактива,
easing(overshoot) для декора. Анимируем важное, остальное статично («переанимация» убивает премиальность).
**`prefers-reduced-motion` обязателен** — отключает декоративные анимации. Только `transform/opacity` (GPU);
`backdrop-blur` ограниченно (ключевые поверхности), не на каждом элементе — чтобы не просесть на слабых/мобильных.

## 4. Токены (единый источник)

`src/lib/theme.js` (экспорт констант) + CSS-переменные в глобальном стиле:
- **Цвета:** `bg/#0a0a0a`, `surface/#141414`, `gold/#d4af37`, `green/#6ee7a8`, `teal/#2dd4bf`, `yellow/#f3d77b`,
  `red/#f8a3a3`, `txt/#fafaf7`, `mut/#a8a8a3`, `dim/#6b6b67` (как сейчас — фиксируем).
- **Motion-токены:** `dur.hover=180ms`, `dur.enter=320ms`, `dur.exit=160ms`, `ease.out=cubic-bezier(.2,.7,.3,1)`,
  `spring={type:'spring',stiffness:400,damping:30}`, `lift=6px`, `scale.hover=1.02`.
- **Радиусы/тени:** `radius.card=16-18px`, `shadow.glow` (золотистая слоёная тень), `shadow.card`.
Существующие `STAGE_META`/`BTN` подключаются к токенам (не дублировать цвета).

## 5. Фон (`DottedBg`/глобальный слой)

Фиксированный фон-слой: dotted-grid (`radial-gradient` 22px) + **золотое aurora** (1-2 размытых золотых пятна,
opacity ≤0.12, медленный дрейф, `prefers-reduced-motion` → статика) + радиальная виньетка по краям. CSS-only.

## 6. Поверхности

- **Glass-карточки:** градиент-фон + `backdrop-filter:blur` (ключевые) + градиент-бордер.
- **Слоёные золотистые тени** на hover (тонированные `rgba(212,175,55,..)`, не серые) → глубина.
- **Frosted-glass sticky-шапка:** `backdrop-blur`+saturation; сохранить существующий layoutId tab-indicator.

## 7. Интеракции — компоненты в `src/components/`

- `SpotlightCard.jsx` — обёртка-карточка: свечение следует за курсором (CSS-переменная `--mx/--my` через
  `onMouseMove`) + hover-lift (`translateY(-6px)`, `scale(1.02)`, золотистая тень). Базовая поверхность для карточек.
- `MagneticButton.jsx` — кнопка-CTA, притягивается к курсору (spring через framer-motion `useSpring`),
  градиент-золото; на touch/`reduced-motion` — обычная кнопка.
- `CountUp.jsx` — анимация числа (свой хук на framer-motion `animate()` + `Intl.NumberFormat` для ₽/локали;
  БЕЗ платного Motion+). Для KPI/денег.
- `Reveal.jsx` — scroll-reveal обёртка (`whileInView`, fade+slide 8-12px, once).
- `Skeleton.jsx` — shimmer-плейсхолдер (анимированный градиент) вместо «Загружаем…».
Все уважают `useReducedMotion`.

## 8. Cmd+K command palette (`src/components/CommandPalette.jsx`)

`Ctrl/Cmd+K` (глобальный слушатель в App) → модалка-палитра (createPortal, blur-backdrop): поле поиска +
fuzzy-список: проекты (→ открыть карточку), задачи (→ вкладка задач), заказы (→ «Мои заказы»), разделы
(→ переход по табам). Источник данных — уже загруженные `projects/tasks/clientProjects` в App. Навигация
стрелками + Enter; Esc закрыть. Лёгкая (без внешних cmdk-библиотек — свой минимальный фильтр).

## 9. Применение по экранам (App.jsx подключает примитивы)

Дашборд → Проекты (карточки) → Задачи (TasksView/доска) → Финансы → Аналитика → Заказчики → «Мои заказы»
→ Отчёты (ReportViewer). Карточки → `SpotlightCard`; KPI/деньги → `CountUp`; пустые состояния → `Skeleton`/
крупная иконка; секции дашборда → `Reveal`; ключевые CTA → `MagneticButton`; шапка → frosted.

## 10. Фазы реализации (один спек, поэтапно)

- **Фаза A (фундамент):** `theme.js` токены + глобальный фон (`DottedBg`: dotted+золотое aurora+виньетка) +
  frosted-шапка + `Skeleton` + слоёные тени на карточках. Базовая «премиальность» сразу видна.
- **Фаза B (интеракции):** `SpotlightCard` + `CountUp` + `Reveal` + `MagneticButton`, применить к экранам.
- **Фаза C (Cmd+K):** `CommandPalette` + глобальный хоткей + источники поиска.

## 11. Критерии верификации

1. `npm run build` зелёный после каждой фазы.
2. `prefers-reduced-motion: reduce` (DevTools emulate) → декоративные анимации (aurora-дрейф, scroll-reveal,
   count-up, magnetic) отключаются/статичны; контент и иерархия читаемы.
3. FPS: на дашборде со множеством карточек скролл/hover плавные (blur не на каждом элементе; transform/opacity).
4. Cmd+K: открывается по Ctrl/Cmd+K, ищет проекты/задачи/заказы/разделы, Enter переходит, Esc закрывает.
5. Мобильная ширина (≤640px): эффекты деградируют корректно (magnetic→обычная кнопка, aurora лёгкая,
   карточки колонкой), вёрстка не ломается.
6. Бренд/функционал не сломаны: золото-акцент, lucide, существующие данные/RPC/фичи (№10, заказчик) целы.

## 12. Вне scope (YAGNI)

- Новый display-шрифт (по умолчанию Geist) — добавить точечно по желанию.
- Светлая тема / переключатель тем.
- 3D-ассеты, тяжёлые иллюстрации (вес).
- Редизайн БД/логики — только визуальный слой.
- WebGL/GSAP — обходимся framer-motion (уже есть).

## 13. Риски и ловушки

- **Монолит App.jsx (~8100 строк) + диск F: сбоит на fsync** при правке больших файлов → новые примитивы
  ОТДЕЛЬНЫМИ файлами (минимум правок монолита); App.jsx править через C-копию (Write на C → Copy-Item C→F
  с ретраями), как в фазе D. **chkdsk F: /f** — давний долг, диск шумит.
- **Производительность:** `backdrop-blur` + aurora + spotlight на многих карточках могут просадить FPS/моб →
  blur ограниченно, aurora 1-2 пятна low-opacity, spotlight через CSS-переменную (не re-render), reduced-motion.
- **framer-motion вес:** уже в бандле — новые эффекты не добавляют зависимостей (свои CountUp/Magnetic на нём).
- **Переанимация:** держать дисциплину §3 (сдержанные тайминги/amplitude), иначе «китч».
- **Среда:** применение/деплой — только по слову «деплой»; build тестировать после каждой фазы.
