# apartment-tracker

Трекер квартир с Telegram-ботом как единственным интерфейсом. Работает локально
на твоём ПК. Когда ПК включён — бот отвечает и шлёт уведомления о новых лотах
и снижении цен. Когда выключен — пауза.

**Что умеет:**
- 🔍 Парсит **CIAN** автоматически по сохранённым URL-фильтрам.
- 📝 Принимает **Yandex Realty** через ручной ввод полей или HTML-файл (Яндекс блокирует ботов капчей — автомат работает не всегда).
- 📊 Считает **скоринг** 0-100 по твоим весам (цена/м², метро, этаж, ремонт и т.д.).
- 📉 Ведёт **историю цен** и шлёт уведомления при изменениях.
- 📑 Экспортирует **Excel-дашборд** по запросу.
- 🤖 Всё это — через команды боту в Телеграме.

## Установка (Windows, для не-программиста)

### 1. Установи Python (если ещё нет)

1. Скачай Python 3.10+ с https://www.python.org/downloads/
2. При установке поставь галку **«Add python.exe to PATH»**.
3. Готово. Закрой installer.

### 2. Скачай проект

Самый простой способ:
1. Открой страницу проекта на GitHub.
2. Нажми зелёную кнопку **Code** → **Download ZIP**.
3. Распакуй ZIP в удобную папку, например `C:\apartment-tracker\`.

### 3. Запусти `setup.bat` двойным кликом

В папке проекта дважды кликни на **setup.bat**.

Скрипт автоматически:
- Создаст виртуальное окружение Python (`.venv`).
- Поставит все зависимости.
- Создаст `config.yaml` и `.env`.
- Инициализирует базу данных.

Если что-то пойдёт не так — окно не закроется, прочитаешь ошибку.

### 4. Создай Telegram-бота

1. В Телеграме найди бота **@BotFather**. Напиши ему:
   ```
   /newbot
   ```
2. Придумай имя боту (любое) и username (должен заканчиваться на `bot`, например `myhome_finder_bot`).
3. BotFather пришлёт **токен** — длинная строка типа `123456:ABC-DEF1234ghIkl...`. Скопируй.

### 5. Узнай свой chat_id

В Телеграме найди бота **@userinfobot** и напиши ему `/start`. Он пришлёт твой `id` числом.

### 6. Впиши токен и chat_id

Открой файл `.env` в папке проекта **Блокнотом** и заполни:

```
TELEGRAM_BOT_TOKEN=123456:ABC-DEF1234ghIkl...
TELEGRAM_CHAT_ID=123456789
```

Сохрани.

### 7. Запусти бота

Двойной клик на **run.bat**.

Окно командной строки останется открытым — пока оно открыто, бот работает.
Закроешь — бот остановится. Это нормально.

### 8. Открой своего бота в Telegram

Найди по username которое ты ему дал в шаге 4. Напиши `/start`. Бот ответит.

## Команды бота

### Лоты

- `/add <URL>` — добавить квартиру. Если ссылка CIAN — может предложить заполнить вручную (для одного лота). Если Yandex Realty — попытается распарсить, при провале спросит поля по очереди.
- `/list` — топ-10 квартир по скорингу.
- `/score` — пересчитать скоринг (например после изменения весов в config.yaml).
- `/export` — пришлёт Excel-файл с дашбордом.
- `/history <ID>` — история цены конкретной квартиры. ID берётся из `/list`.
- `/remove <ID>` — пометить квартиру как снятую.

### Фильтры (только CIAN)

Фильтр — это URL поиска на CIAN с твоими параметрами (район, цена, комнаты). Бот будет регулярно прогонять этот URL и присылать новые карточки.

- `/filter_add` — добавить фильтр. Бот спросит имя и URL.
- `/filter_list` — все сохранённые фильтры.
- `/scrape` — прогнать все включённые фильтры прямо сейчас.

Бот сам прогоняет включённые фильтры **раз в час** (настраивается в `config.yaml` — `scrape_interval_seconds`).

### HTML-файл от Yandex Realty

Если автопарсинг Yandex упал в капчу:
1. Открой карточку в браузере.
2. **Ctrl+S** → сохрани как «Веб-страница, только HTML».
3. Перешли HTML-файл боту обычным вложением.

Бот извлечёт что сможет (цена, площадь, этаж, метро) и спросит остальное.

### Прочее

- `/status` — сколько лотов, фильтров, где БД.
- `/help` — все команды.
- `/cancel` — отменить текущую форму (если запутался в /add или /filter_add).

## Конфиг (config.yaml)

В корне проекта файл `config.yaml`. Ключевое:

### Веса скоринга

```yaml
scoring:
  weights:
    price_per_m2: 25      # цена за м² — самое важное
    metro_distance: 15    # минут пешком до метро
    floor: 10             # не 1-й, не последний
    area_total: 10        # попадает ли площадь в желаемый диапазон
    renovation: 10
    year_built: 10
    rooms: 10             # точное совпадение по комнатам
    seller_type: 5        # собственник > агент
    photos_count: 5       # больше фото — выше доверие
```

Сумма должна быть 100. Меняй веса под себя, потом в боте `/score`.

### Пороги

```yaml
scoring:
  thresholds:
    price_per_m2:
      best: 200000        # ₽/м² и ниже = идеально (1.0)
      worst: 500000       # ₽/м² и выше = ужасно (0.0)
    metro_distance:
      best: 5             # ≤5 мин = идеально
      worst: 30
    area_total:
      ideal_min: 40       # ВНУТРИ диапазона = идеально
      ideal_max: 80
    rooms:
      target: 2           # хочу 2-комнатную; 1 или 3 = 0.5, остальное = 0
```

### Уведомления

```yaml
telegram:
  notify_on:
    new_listing: true
    price_drop: true
    price_increase: false
    status_change: true
    score_above: 70       # уведомлять только если скоринг ≥ 70 (0 = всегда)
  scrape_interval_seconds: 3600   # как часто авто-прогон, секунды
```

## Что НЕ умеет (честно)

- **Yandex Realty без капчи не работает стабильно.** Яндекс блокирует ботов. Реальный путь — ручной ввод или HTML-файлы. Если хочешь полную автоматизацию по Яндексу — нужен headless-браузер с решением капчи (тяжелее, в этой версии не реализовано).
- **Avito** не подключен.
- **Карт и расстояний** до конкретных точек (работа, школа) автомат не считает — поле «минут до метро» вводится с самой карточки.
- **Несколько пользователей** — нет. Это персональный инструмент: один бот = один владелец (whitelist по chat_id).
- **Когда ПК выключен** — бот не работает (это сознательное решение, по твоему запросу).

## Как это устроено внутри (для интересующихся)

```
apartment-tracker/
├── setup.bat / setup.sh      # установка одним кликом
├── run.bat / run.sh          # запуск бота
├── config.yaml               # твои настройки
├── .env                      # секреты (токен, chat_id)
├── data/apartments.sqlite    # БД (создастся автоматом)
├── exports/                  # Excel-выгрузки
└── apartment_tracker/
    ├── bot.py                # сам Telegram-бот (главный файл)
    ├── cli.py                # альтернативный CLI (если бот не нравится)
    ├── db.py                 # SQLite
    ├── scoring.py            # скоринг
    ├── exporter.py           # Excel
    └── sources/              # парсеры CIAN, Yandex, ручной
```

- **БД** — SQLite, один файл `data/apartments.sqlite`. Можно открыть любым SQLite-viewer (например DB Browser for SQLite) и посмотреть таблицы.
- **Скоринг** — линейная нормализация по порогам из конфига; для категориальных полей (ремонт, продавец) — словари в конфиге.
- **Дедупликация** — по `source:external_id` (например `cian:12345678`). Один и тот же лот не задублируется, при повторных встречах обновляется `last_seen` и добавляется новая точка цены если она изменилась.

## Обновление весов скоринга

1. Открой `config.yaml`, поменяй веса/пороги.
2. В боте: `/score` — пересчитает все лоты с новыми весами.
3. `/list` — увидишь новое ранжирование.

История скорингов **не** сохраняется (только текущий). Если важно видеть как менялся рейтинг — экспортируй в Excel **до** и **после** изменения весов.
